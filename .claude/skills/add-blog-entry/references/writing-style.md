# Yusuke の文体ガイド

過去のブログ・career・values を分析した文体の特徴。

## トーン

- **です・ます調 / だ・である調はその時の気分で変わる**。ユーザーのメッセージの雰囲気から読み取り、迷ったら質問する
- 断定は避け「〜だと思っている」「〜と考えている」で締める
- 押しつけがましくない、謙虚なトーン

## 文の長さ・構造

- 文は短め。長文は意識的に区切る
- 「なので、」「そのため、」「ただ、」などの接続詞でリズムを作る
- 括弧書きで補足コメントを添えることがある（でも多用しない）

## 内容の組み立て方

- **体験ベース**で書く。「〜をやってみた」「〜に気づいた」から始まることが多い
- 抽象的な主張には必ず具体的なエピソードを添える
- 外部リンクを積極的に使って、詳細は既存記事に委ねる
- 見出しで構造化する（長い記事のとき）

## よく使う表現

- 「〜だと思います」「〜と考えています」
- 「なので、〜」「そのため、〜」
- 「〜だったのですが、〜」（逆接で展開）
- 「せっかく〜するなら〜したい」

## 避けること

- 「！」などの強調記号の多用
- 「絶対」「必ず」などの断定的な言い切り
- 読者を置いてけぼりにする専門用語
- 長い段落（4〜5行を超えたら改行を入れる）

## 文体サンプル

> 人生の大半は仕事に時間を使います。なので、せっかくやるなら良い思い出になる仕事をしたいです。

> チームにヒーローやカリスマは必要ありません。そういう状況はチームにとって健全ではないと考えています。

> 今までずっとエンジニア文化の強い会社にいて、居心地が良かったのですが、事業として成長できている感覚が薄くなっていました。ただ、そうは言ってもビジネスの文化が強い会社が良いとも思わず、両者の良いとこを尊重し合える組織で働きたいと思っていました。
