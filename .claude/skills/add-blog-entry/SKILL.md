# add-blog-entry

`src/content/notes/` に Markdown ファイルを作成し、yusuke ページの blog タブに表示されるメモを追加する。

## フロー

### Step 1: 情報収集（質問）

まずユーザーに質問を投げ、ドラフトに必要な情報を集める。すでにメッセージに含まれている情報は聞き直さない。

聞くべきこと（足りない分だけ）:
- **何について書くか** — テーマ・出来事・気づき
- **誰に向けて / どんな文脈か** — 自分用メモ？誰かに伝えたい？
- **一番言いたいこと** — 結論・主張・感想

### Step 2: ドラフト作成

`references/writing-style.md` を読み、Yusuke の文体に合わせてドラフトを書く。

集めた情報をもとに **400文字前後** の日本語ドラフトを書く。

- 語調（です・ます / だ・である）はユーザーのメッセージの雰囲気から判断、迷ったら確認する
- 体験ベース、短い文、謙虚なトーン（詳細は writing-style.md 参照）
- 構成は自由。箇条書きでも文章でも
- タイトル案も一緒に提示する

ドラフトをチャットに出力し、「このまま保存しますか？修正があれば教えてください。」と伝える。

### Step 3: 確認・修正

ユーザーが OK を出すか、修正内容を伝えてきたら内容を反映する。

### Step 4: ファイル作成

**保存先:** `src/content/notes/YYYY-MM-DD-slug.md`

slug のルール:
- タイトルから英語に意訳してハイフン区切り（例: `team-culture`、`astro-content-collections`）
- date は今日の日付をデフォルトにする

**テンプレート:**

```md
---
date: "YYYY-MM-DD"
title: "タイトル"
---

本文
```

作成後、ファイルパスと閲覧 URL（`/yusuke/notes/[slug]`）を伝える。`npm run build` は実行しない。
