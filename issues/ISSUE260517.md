## 今回の調査で追加（2026-05-17）

### ~~10. `Entry.tsx` がデッドコード~~ ✅ 解消済み

**ファイル**: `src/yusuke/client/content/Entry.tsx`

どこからも import されていない。`Card` コンポーネントで同等の機能が実装済み。削除して問題なし。

### ~~11. `children: any` 型が複数箇所に存在~~ ✅ 解消済み

**ファイル**:
- `src/yusuke/client/modal-context.tsx:31`
- `src/yusuke/client/components/ModalDialog.tsx:4`
- `src/yusuke/client/components/Tabs.tsx:12`

`children` および render prop の戻り値が `any` 型。`JSX.Element` または `ComponentChildren`（Hono の型）に置き換えるべき。

### ~~12. `BlogContent.tsx` でオブジェクトのキーをソートしている~~ ✅ 解消済み

**ファイル**: `src/yusuke/client/content/blog/BlogContent.tsx:16-26`

`Record<string, BlogEntry[]>` に年をキーとして格納し、`Object.keys(...).sort().reverse()` で降順ソートしている。JavaScript オブジェクトのキー順序は文字列キーでも保証されないケースがある。`Array<{ year: string; entries: BlogEntry[] }>` に変更すべき。

### ~~13. `modal-context.tsx` で React state と DOM 直接操作が混在~~ ✅ 解消済み

**ファイル**: `src/yusuke/client/modal-context.tsx:38-53`

`modalState.isOpen` で状態管理しつつ、`dialogElement.showModal()` / `dialogElement.close()` を直接呼んでいる。state が変わっても DOM 操作が失敗した場合に不整合が生じる可能性あり。`<dialog>` の open 状態は DOM に持たせて React state と分離するか、`ref` 経由で一元管理すべき。

### ~~14. `Timeline.tsx` の定数が CSS と手動同期~~ ✅ 解消済み

**ファイル**: `src/yusuke/client/components/Timeline.tsx:6-7`

`CARD_HEIGHT = 200`、`CARD_MARGIN = 20` をグリッド行計算に使用しているが、実際の表示サイズは Tailwind クラス（`p-2 sm:p-3` 等）で決まる。CSS を変更しても定数は更新されず、タイムラインのレイアウトが崩れる。

### ~~15. `utils.ts` に未使用関数が存在~~ ✅ 解消済み

**ファイル**: `src/yusuke/client/content/utils.ts:38-60`

`calculateDuration`、`getDurationMs` が定義されているが、コード内で呼ばれていない。削除してよい。

### ~~16. `PersonalContent.tsx` の `url` / `detail` 両立時に `detail` が無視される~~ ✅ 解消済み

**ファイル**: `src/yusuke/client/content/personal/PersonalContent.tsx:36-61`

`if (entry.url)` が先にチェックされ、そこで `return` するため、`url` と `detail` が両方あるエントリは `detail` が無視される。意図的な仕様なら明示すべき。

`detail` を優先してモーダル表示、`url` がある場合はカード下に「サイトを開く」リンクを表示。`GalleryEntry.detail` に JSDoc を追記。
